- 한글 명세/설계 파일들은 각 기능, UI, 요구사항, 모듈설계 등 실전 운용 기준으로 작성됨
- **modules.zip** 등 압축 모듈은 실제 BAS에서 동작 가능한 고성능 모듈만 포함(더미금지)
- XML 출력결과는 각 저장소별로 실전 배포 경로에 생성(700MB 이상, 7,170개 기능, 300,000+ 액션)
- 중복 기능은 고성능/최적화 버전만 채택, 테스트용/더미 기능은 포함되지 않음

   → 한글 명세/설계 파일 파싱 → 실제 모듈/로직 연결 → XML 700MB+ 생성

3. **상업용 실전 배포**  
   → 더미/중복 기능 제거, 고성능 기능만 채택
   → 실제 UI·모듈·로직과 연결되는 구조
   → 각 저장소별로 실전 XML 결과물 배포

1. BAS 29.3.1 API 예시
BAS는 시각적 블록(노드) 기반 자동화이지만, 내부적으로 JavaScript 및 자체 API 호출을 지원합니다.

예시: 브라우저 자동화 API
JavaScript
// BAS 내장 API 예시: 페이지 열기, 데이터 추출
function main() {
    // 브라우저 열기
    BAS_Open("https://example.com");

    // 텍스트 추출
    let title = BAS_Extract("//title", "xpath");
    BAS_Log("페이지 제목: " + title);

    // 폼 입력
    BAS_SetValue("//input[@name='search']", "검색어", "xpath");
    BAS_Click("//button[@type='submit']", "xpath");

    // 결과 추출
    let results = BAS_Extract("//div[@class='result']", "xpath");
    BAS_Log("검색 결과: " + results);

    // 브라우저 닫기
    BAS_Close();
}
주요 API 함수(블록) 구조
BAS_Open(url): URL로 브라우저 열기
BAS_Extract(selector, method): XPath/CSS 등으로 값 추출
BAS_SetValue(selector, value, method): 입력값 설정
BAS_Click(selector, method): 클릭
BAS_Log(message): 로그 출력
BAS_Close(): 브라우저 닫기

. XML 안에 JSON이나 HTML을 넣을 수 있나요?
 • 넣을 수는 있습니다. XML은 구조화된 데이터를 표현하는 포맷이기 때문에, JSON이나 HTML을 문자열로 변환하여 XML 요소 안에 포함시킬 수 있습니다.
 • 하지만 이것이 필수는 아닙니다. 시스템에서 요구하는 명세(spec)에 따라 XML 구조 안에 JSON이나 HTML이 포함될지 여부가 달라집니다.
 예를 들어: <data>
    <json>{"key": "value"}</json>
    <html>&lt;div&gt;Hello&lt;/div&gt;</html>
</data>
위와 같이 JSON이나 HTML을 XML 안에 문자열로 넣을 수 있습니다.


. 로고는 어떻게 처리하나요?
 • 보통은 이미지 파일 경로(URL)만 XML에 포함시키는 것이 일반적입니다.
 • 만약 로고 이미지를 XML 안에 “직접” 포함시켜야 한다면, Base64 인코딩을 사용하여 이미지 데이터를 문자열로 변환하고 XML에 포함시킵니다.
 예시: <logo>data:image/png;base64,iVBORw0KGgoAAAANSUhEUg...</logo>

시스템 요구 사양(spec)에 따라 다릅니다.
 • 요구 사양에서 JSON이나 HTML을 XML 안에 포함시키도록 명시되었다면 넣어야 하지만, 그렇지 않다면 필수는 아닙니다.
 • 로고 역시 URL을 사용하는지, Base64로 인코딩하여 XML에 포함하는지는 해당 시스템의 요구사항에 따라 결정됩니다.
 즉, “무조건 다 넣어야 한다”는 것은 아니며, 요구 사양에 맞게 작성하는 것이 중요합니다.

. XML 안에 JSON이나 HTML을 넣을 수 있나요?
 • 넣을 수는 있습니다. XML은 구조화된 데이터를 표현하는 포맷이기 때문에, JSON이나 HTML을 문자열로 변환하여 XML 요소 안에 포함시킬 수 있습니다.
 • 하지만 이것이 필수는 아닙니다. 시스템에서 요구하는 명세(spec)에 따라 XML 구조 안에 JSON이나 HTML이 포함될지 여부가 달라집니다.
 예를 들어: <data>
    <json>{"key": "value"}</json>
    <html>&lt;div&gt;Hello&lt;/div&gt;</html>
</data>
위와 같이 JSON이나 HTML을 XML 안에 문자열로 넣을 수 있습니다.


. 로고는 어떻게 처리하나요?
 • 보통은 이미지 파일 경로(URL)만 XML에 포함시키는 것이 일반적입니다.
 • 만약 로고 이미지를 XML 안에 “직접” 포함시켜야 한다면, Base64 인코딩을 사용하여 이미지 데이터를 문자열로 변환하고 XML에 포함시킵니다.
 예시: <logo>data:image/png;base64,iVBORw0KGgoAAAANSUhEUg...</logo>

시스템 요구 사양(spec)에 따라 다릅니다.
 • 요구 사양에서 JSON이나 HTML을 XML 안에 포함시키도록 명시되었다면 넣어야 하지만, 그렇지 않다면 필수는 아닙니다.
 • 로고 역시 URL을 사용하는지, Base64로 인코딩하여 XML에 포함하는지는 해당 시스템의 요구사항에 따라 결정됩니다.
 즉, “무조건 다 넣어야 한다”는 것은 아니며, 요구 사양에 맞게 작성하는 것이 중요합니다.
